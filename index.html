O caminho correto para o desenvolvimento front-end.

A WordWideWeb foi um projeto que buscou conectar usuários para que eles pudessem trocar conteúdo de hipertexto. Felizmente o projeto tomou proporções gigantes em relação ao seu objetivo inicial.

A web hoje é uma das principais plataformas de desenvolvimento.

Aplicações web cara vez mais ganham mercado. No servidor temos inúmeras linguagens para trabalhar, como Java, Ruby, Python, etc.. Mas apenas três linguagens interagem com os navegadores: HTML, CSS e JavaScript

Resumindo: precisamos aprender bem essas três linguagens para criar aplicações escaláveis, performáticas e que atendam o maior número de usuários.

Por onde começo? Não é só usar frameworks, bibliotecas ou plugins?

Não.

Na web há diversas páginas com o lado do cliente possuindo um grande número de problemas, otimizações a serem feitas e pouco performáticas.

80-90% do tempo de carregamento de uma página é gasto no front-end. (Steve Sounders, 2007).

Todo mundo sabe disso. Mas como vamos resolver? São muitos caminhos a serem seguidos.

Talvez a saída seja irmos na raíz do problema e ensinar padrões ao invés de ir diretamente para frameworks ou bibliotecas.

Hoje temos um problema com excesso de informação. Mesmo com bastante recursos, fica difícil saber o mais simples que é por onde começar. É isso que vamos mostrar aqui, o caminho correto para um bom desenvolvimento front-end.
